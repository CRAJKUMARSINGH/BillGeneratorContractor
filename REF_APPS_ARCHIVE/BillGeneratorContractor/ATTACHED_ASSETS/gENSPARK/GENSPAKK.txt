## 🔬 ROOT CAUSE DIAGNOSIS — WHY IT FAILS

The current `modules/pwd_schedule_parser.py` uses this broken approach:

```python
# THE PROBLEM: This only detects clean, straight table grid lines
kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (40, 1))
detect = cv2.morphologyEx(img, cv2.MORPH_OPEN, kernel)
# Then filters: w > 600 and h > 30  ← FAILS on WhatsApp photos
```

**WhatsApp JPEG photos are compressed** (85% quality), possibly skewed/rotated, with blurry table lines. The morphological kernel can't find clean horizontal lines, so 0-1 rows are detected instead of 15-20.

The previous engineer then **wrongly concluded** "free OCR can't do this." That was **completely incorrect**.
