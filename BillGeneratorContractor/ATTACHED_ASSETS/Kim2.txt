Document Overview
Title: BillGeneratorContractor -- Foolproof OCR Enhancement Guide
Author: Er. Rajkumar Singh Chauhan
Purpose: Robust OCR + validation architecture for converting PWD Schedule-G Work Order images and quantity text files into standard Excel input
1.  Core Objective
Problem: The billing engine works correctly with standard Excel input, but the input builder (converting Work Order images + Quantity files to Excel) is unreliable.
Goal: Create a highly reliable extraction system that never produces a wrong bill silently. If extraction fails, the software must stop and alert the user.
----
2.  System Architecture Pipeline
IMAGE → Image Preprocessing → Table Grid Detection → Row Extraction →
OCR Row Processing → Item Code Extraction → Quantity Matching →
Validation Layer → Excel Generation
----
3.  OCR Reliability Strategies
Strategy	Description
Multi-Mode OCR	Uses PSM 6 (uniform text), PSM 4 (columns), PSM 11 (sparse text)
OCR Error Correction	Auto-fixes common mistakes: O→0, l→1, S→5
Row-Based OCR	Detects table grid first, then OCR each row separately (dramatically improves accuracy for structured documents)
----
4.  Key Implementation Components
Image Preprocessing
•  Convert to grayscale
•  Gaussian blur (5,5 kernel)
•  Adaptive thresholding (Gaussian C, block size 11, C=2)
Table Grid Detection
•  Morphological opening with rectangular kernel (40,1)
•  Contour detection with filtering (width > 600, height > 30)
•  Sort rows by Y-coordinate
OCR Row Extraction
•  Crop each detected row
•  Tesseract OCR with OEM 3, PSM 6
•  Apply error corrections (O→0, l→1)
Item Code Extraction
•  Pattern: \d+.\d+.\d+ (e.g., 1.1.1, 1.1.2, 1.1.3)
•  PWD Schedule-G codes are extremely reliable identifiers
----
5.  Data Parsing
Work Order Items
Extracts per row:
•  Item code (regex match)
•  Description (text after code)
•  Unit (4th element from end)
•  Rate (3rd element from end)
•  Quantity (2nd element from end)
Quantity File Format
1.1.1=2
1.1.2=4
1.1.3=2
Simple key-value pairs: code=quantity
6.  Critical Validation Layer
The system must never generate a bill if inconsistencies are found.
Validation checks:
•  ✓ Work order items detected
•  ✓ Quantity file items exist
•  ✓ Codes match between work order and quantity file
•  ✓ Rates are numeric
Example validation:
def validate_qty(qty_dict, codes):
missing = set(qty_dict.keys()) - set(codes)
if missing:
raise Exception(f"Unknown item codes in qty file: {missing}")

UNIT	Measurement unit
RATE	Unit rate
QTY	Quantity
AMOUNT	Calculated (RATE × QTY)
